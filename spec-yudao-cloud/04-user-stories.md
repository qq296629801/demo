# 04 — 用户故事

## 角色定义

| 角色 | 说明 |
|------|------|
| 系统管理员 | 负责用户/权限/租户/系统配置的管理员 |
| 普通员工 | 使用各业务功能的内部员工 |
| 销售人员 | 使用 CRM 进行客户跟进的销售 |
| 采购/仓库人员 | 使用 ERP/WMS 进行进销存管理的人员 |
| 客户/会员 | 使用商城下单的终端消费者 |
| 开发者 | 基于 yudao-cloud 进行二次开发的工程师 |

---

## 用户故事列表

### 系统管理

| 编号 | 角色 | 故事 | 验收标准 |
|------|------|------|---------|
| US-001 | 系统管理员 | 我希望为新员工创建系统账号并分配角色，以便其能访问对应功能 | 账号创建成功，登录后仅显示角色对应的菜单和按钮 |
| US-002 | 系统管理员 | 我希望批量导入用户，以便快速完成组织架构初始化 | Excel 模板下载→填写→上传，成功导入并显示导入结果 |
| US-003 | 系统管理员 | 我希望配置多个租户，以便平台支持 SaaS 多租户部署 | 不同租户数据完全隔离，管理员只能看到本租户数据 |
| US-004 | 系统管理员 | 我希望查看所有接口的访问日志，以便排查问题和审计操作 | API 访问日志记录请求/响应，支持按用户/时间/接口筛选 |
| US-005 | 普通员工 | 我希望使用手机验证码快速登录，以便不用记住密码 | 输入手机号→获取验证码→登录成功，返回 Token |

### 工作流

| 编号 | 角色 | 故事 | 验收标准 |
|------|------|------|---------|
| US-006 | 普通员工 | 我希望在线提交请假申请，以便不用找领导签字 | 填写假期类型/时间/原因→提交→流程流转到部门经理审批 |
| US-007 | 普通员工 | 我希望看到我发起的所有流程状态，以便跟踪审批进度 | 我的流程列表显示状态：审批中/通过/拒绝/撤销 |
| US-008 | 系统管理员 | 我希望用可视化设计器设计审批流程，以便不需要写代码配置流程 | BPMN 设计器拖拽连线，保存后可立即发起流程 |

### CRM 销售

| 编号 | 角色 | 故事 | 验收标准 |
|------|------|------|---------|
| US-009 | 销售人员 | 我希望录入潜在客户线索，以便追踪和转化 | 填写线索信息→保存→可在线索列表查看和跟进 |
| US-010 | 销售人员 | 我希望把线索转化为客户，以便进入正式销售流程 | 线索详情点击"转化"→自动创建客户，保留线索来源 |
| US-011 | 销售人员 | 我希望查看我负责的客户统计，以便了解业绩情况 | 统计页显示客户数量、商机总额、回款金额等指标 |
| US-012 | 销售主管 | 我希望查看团队成员的销售漏斗，以便监控销售过程 | 漏斗图显示：线索→客户→商机→合同→回款各阶段数量和金额 |

### 商城

| 编号 | 角色 | 故事 | 验收标准 |
|------|------|------|---------|
| US-013 | 系统管理员 | 我希望上架商品（含多规格 SKU），以便消费者可以下单购买 | 创建 SPU+SKU，设置价格/库存/规格，发布后前端可见 |
| US-014 | 客户/会员 | 我希望使用微信支付下单，以便完成购买 | 选商品→加购物车→结算→微信支付→支付成功→订单状态变为待发货 |
| US-015 | 系统管理员 | 我希望配置秒杀活动，以便通过限时优惠促进销售 | 设置秒杀时间段/参与商品/秒杀价格，活动开始后前端可购买 |
| US-016 | 客户/会员 | 我希望申请订单售后退款，以便在收到不满意商品时维权 | 订单完成后 X 天内可申请售后，管理员审核后自动退款 |
