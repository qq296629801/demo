# 02 — 软件需求规格（SRS）

## 1. 引言

### 1.1 目的

本 SRS 描述 yudao-cloud 开源企业级微服务快速开发平台的功能需求，覆盖 14 个业务模块，367 张数据库表。

### 1.2 系统范围

| 模块 | 核心功能概述 |
|------|------------|
| system | 用户权限、角色菜单、字典、租户、通知、日志 |
| infra | 代码生成、文件管理、系统配置、定时任务 |
| bpm | BPMN 流程设计、实例管理、审批任务、OA 请假 |
| ai | 大模型接入、聊天对话、AI 绘画/音乐/写作 |
| crm | 销售全流程：线索→客户→商机→合同→回款 |
| erp | 采购→生产→销售→库存→财务全链路 |
| iot | 物联网设备管理、数据采集、规则告警 |
| pay | 聚合支付、退款、钱包、转账 |
| mall | 电商商城：商品/订单/营销/统计 |
| wms | 仓储：入库/出库/调拨 |
| mp | 微信公众号全功能管理 |
| member | 会员注册/积分/等级/标签 |
| report | 可视化报表（Goview） |

## 2. 功能需求

### FR-001 系统管理（yudao-module-system）

#### FR-001-01 用户管理

| 功能点 | 接口 | 权限码 |
|--------|------|--------|
| 创建用户 | POST /system/user/create | system:user:create |
| 修改用户 | PUT /system/user/update | system:user:update |
| 删除用户 | DELETE /system/user/delete | system:user:delete |
| 批量删除 | DELETE /system/user/delete-list | system:user:delete |
| 重置密码 | PUT /system/user/update-password | system:user:update-password |
| 修改状态 | PUT /system/user/update-status | system:user:update |
| 分页列表 | GET /system/user/page | system:user:query |
| 导出 Excel | GET /system/user/export-excel | system:user:export |
| 导入 Excel | POST /system/user/import | system:user:import |

#### FR-001-02 认证授权

| 功能点 | 接口 | 说明 |
|--------|------|------|
| 账号密码登录 | POST /system/auth/login | 返回 AccessToken + RefreshToken |
| 短信登录 | POST /system/auth/sms-login | 手机验证码登录 |
| 社交登录 | POST /system/auth/social-login | 微信/QQ/GitHub 等 |
| 退出登录 | POST /system/auth/logout | 清除 Token |
| 刷新 Token | POST /system/auth/refresh-token | 使用 RefreshToken 续期 |
| 获取权限菜单 | GET /system/auth/get-permission-info | 返回用户菜单树和权限码集合 |

#### FR-001-03 角色/菜单/权限

- 角色 CRUD + 分配菜单权限（`/system/role/**`）
- 菜单 CRUD + 树形结构（`/system/menu/**`）
- 用户分配角色（`/system/permission/assign-user-role`）

#### FR-001-04 多租户

- 租户 CRUD + 套餐管理
- 租户域名绑定、过期时间、开关状态
- 数据行级隔离，框架自动注入 `tenant_id`

#### FR-001-05 其他

- 字典管理（类型 + 数据）
- 部门管理（树形）
- 岗位管理
- 通知公告（系统通知 + 站内信）
- 操作日志 / 登录日志
- 短信渠道 + 模板管理
- 邮件账号 + 模板管理
- OAuth2 客户端管理

|--------|------|

| 数据源配置 | 动态多数据源管理 |

### FR-005 CRM（yudao-module-crm）

销售漏斗全流程：

```
线索(Clue) → 客户(Customer) → 联系人(Contact) → 商机(Business) → 合同(Contract) → 回款(Receivable)
```

| 功能点 | 说明 |
|--------|------|
| 线索管理 | 线索录入/公海/认领/转化为客户 |
| 客户管理 | 客户 CRUD、公海配置、画像分析 |
| 联系人 | 客户联系人 CRUD、与商机关联 |
| 商机 | 商机跟进、状态流转、产品报价 |
| 合同 | 合同签订、产品明细、审批流程 |
| 回款 | 回款计划、回款记录 |
| 数据权限 | 按归属人/部门/全公司多维度 |
| 统计报表 | 客户统计、销售漏斗、员工业绩、排行榜 |

### FR-008 商城（yudao-module-mall）

- SPU（商品） + SKU（规格）管理

#### 营销中心

| 秒杀 | 秒杀活动 + 时间段配置 + 秒杀商品 |
| 拼团 | 拼团活动 + 参团记录 |
| 砍价 | 砍价活动 + 帮砍记录 |
| 积分活动 | 积分商品兑换 |

| DIY 装修 | 页面装修 + 模板管理 |
| 客服 | 客服会话 + 消息管理 |

#### 统计中心
- 商品统计、交易统计、会员统计、支付统计

## 3. 非功能需求

| 类别 | 要求 |
|------|------|
| 可用性 | 微服务独立部署，单模块故障不影响其他模块 |
| 性能 | 分页接口 < 500ms（万级数据，索引完善） |
| 安全 | JWT 无状态认证 + RBAC 权限控制 + 数据权限过滤 |
| 多租户 | 数据行级隔离，框架透明注入 |
| 可扩展 | 各模块独立 API 子模块，接口层可单独复用 |
| 可观测 | SkyWalking APM + ELK 日志 + Prometheus 指标 |
