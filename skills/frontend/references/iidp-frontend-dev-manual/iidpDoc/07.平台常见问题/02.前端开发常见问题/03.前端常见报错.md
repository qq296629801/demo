## Cannot set properties of null (setting 'type')

![img](../images/03/03_setting_type.png)

该报错是由于消息提示方法（_ELEMENT.Message_）接收了 `null` 值导致的。

```javascript
window.ELEMENT.Message.error(null);
```

请检查前端拓展内部是否正确传参，以及后端接口返回的数据是否符合 [平台规范](/pages/0b9b0c/) 。

## 平台基础视图未能正确渲染，页面加载后呈空白状态，前端控制台有报错

```javascript
Uncaught TypeError: Cannot read properties of undefined (reading 'items')
// 或
Uncaught TypeError: Cannot read properties of undefined (reading 'body')
```

原因：`loadView` 接口返回的视图信息缺失，或者当前用户没有权限访问该视图。  
解决方法：  
请检查 `loadView` 接口返回的数据。

- loadView 未返回视图信息

   <!-- ![img](../images/03_loadview_no_data_views.png) -->
   <img src="../images/03/03_loadview_no_data_views.png" height="260" />

  **参考 [文档](/pages/f88161/#_1-7-loadview-%E8%BF%94%E5%9B%9E%E7%BB%93%E6%9E%9C%E4%B8%BA-null-%E6%95%B0%E6%8D%AE%E9%94%99%E8%AF%AF%E6%88%96%E6%8A%A5%E9%94%99) 步骤重置种子数据。**

- loadView 返回了视图信息，但对应的 authInfo 为 false

   <!-- ![img](../images/03_loadview_authinfo_false.png) -->
   <img src="../images/03/03_loadview_authinfo_false.png" height="260" />

  **去 用户管理菜单 进行对应视图的授权。**
