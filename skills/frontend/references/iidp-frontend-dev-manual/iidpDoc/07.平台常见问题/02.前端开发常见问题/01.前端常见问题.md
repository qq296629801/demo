## 本地前端项目更新 iidp 平台依赖版本指南

> 涉及的前端依赖包为 **`@tech/t-core`**、**`@tech/t-el-ui`**、**`@tech/t-build`**、**`@tech/t-base`**

### 更新至最新版本

当需要同步平台发布的最新特性或修复补丁时，请执行封装好的更新脚本。

1. 在前端项目 `package.json` 文件中的 `scripts` 处，找到对应的命令。

   ```json
   "update:uat": "npm i @tech/t-core@uat @tech/t-el-ui@uat @tech/t-build@uat @tech/t-base@uat -S --registry http://iidp.chinasie.com:9999/maven/repository/npm-group/",
   "update:HotFix": "npm i @tech/t-core@HotFix @tech/t-el-ui@HotFix @tech/t-build@HotFix @tech/t-base@HotFix -S --registry http://iidp.chinasie.com:9999/maven/repository/npm-group/",
   ```

   > 注：update:xxx 请替换为具体的业务模块后缀（如 update:tech），具体定义请参照项目已有的脚本进行替换。

2. 在项目根目录下运行对应的脚本即可。

   ```bash
   npm run update:xxx
   ```

### 更新至指定版本

1. 修改依赖配置  
    在 `package.json` 的 `dependencies`（注意全小写）节点中找到目标依赖：  
    移除 版本号前的 `^` 符号（避免模糊匹配），替换为目标指定版本号。

   ```json
   // 修改前
   "@tech/t-base": "^2.8.2-HotFix.30",

   // 修改后（移除 ^，填入确切版本）
   "@tech/t-base": "2.8.2-HotFix.24",
   ```

2. 执行安装脚本

   ```bash
   npm run install:tech
   ```

3. 安装成功后，版本号会自动恢复，请检查 `package.json`。
   - **现象**：依赖的版本号前会自动重新添加 `^` 符号。
   - **解释**：这是 NPM 安装时的默认行为，表示已成功安装了你指定的版本，并在未来允许该版本的小幅补丁更新。**此为正常现象，无需再次手动删除。**

::: tip 更新完成后，使用 npm run start 重新启动前端本地服务。
:::

## form 视图，字段配置条件是否必填

> 目前平台不支持直接动态配置字段是否必填，但可通过以下方式实现：

**示例**：根据 `selectStatus` 单选框的值，动态设置 `remark` 字段是否必填。  
实现思路： 
1. 条件字段的变更事件：条件字段（如 `selectStatus`）的值变化时，根据值动态修改目标字段（如 `remark`）的必填状态。
2. 目标字段的生命周期：在目标字段的 `mounted` 生命周期中，根据条件字段的当前值初始化必填状态。

::: details 查看代码

```javascript
columns: [
  {
    displayName: 'selectStatus',
    name: 'selectStatus',
    bind_on_changeHandler: (params) => {
      // debugger;
      let { self: vm, value } = params;
      // 获取表单节点
      const formNode = vm.$select('sdk_test_role_form_main_detail_top_common');
      const remark = vm.$select(
        'sdk_test_role_form_main_detail_top_common_items_0_items_remark'
      );
      if (formNode && remark) {
        if (value === '1') {
          remark.data.disabled = false;
          remark.data.required = true;
          remark.data.rules = [
            {
              required: true,
              trigger: 'change',
              message: '备注为必填项'
            }
          ];
        } else {
          remark.data.disabled = true;
          remark.data.required = false;
          remark.data.rules = [];
          formNode.$ds.form.remark = '';
        }
      }
    }
  },
  {
    displayName: '备注',
    name: 'remark',
    mounted: (vm) => {
      // debugger;
      const formNode = vm.$select('sdk_test_role_form_main_detail_top_common');
      let selectStatus = formNode?.$ds?.form?.selectStatus;
      if (selectStatus === '1') {
        vm.data.disabled = false;
        vm.data.required = true;
        vm.data.rules = [
          {
            required: true,
            trigger: 'change',
            message: '备注为必填项'
          }
        ];
      } else {
        vm.data.disabled = true;
        vm.data.required = false;
        vm.data.rules = [];
        formNode.$ds.form.remark = '';
      }
    }
  }
],
```

:::

若需根据其他多个字段的值，动态设置必填状态，可使用前端拓展 watch 监听字段变更，搭配初始化逻辑实现。
> 参考文档：[watch 监听字段变更](/pages/7e278f/#watch-监听-表单项-相关数据变更-触发自定义逻辑)

  ![img](../images/01/01_loadview_fields.png)

- 检查 column 是否重复，保证字段唯一

### 4. loadView 字段 `authInfo` 返回了 false

- 后端引擎未正确加载该字段，参考后端文档修改。

### 5. 模型内该字段定义了不展示

- 修改字段的 `"display"/"hidden"` 配置为 `true`

### 6. 平台的默认字段配置后不显示

> 平台基础字段（`create_user`、`create_date`、`update_user`、`update_user` 等），后端引擎内定义了 `默认不展示`

- 视图 column 字段处，配置字段显示：

```json
"display": true,
"custom": true,
```

### 7. 用户偏好影响，loadView 接口返回的 advancedComponent 内有内容

- 参考 [清除 advancedComponent](/pages/a354d5/#%E8%A7%86%E5%9B%BE%E5%AD%97%E6%AE%B5%E6%B8%B2%E6%9F%93%E4%B8%8E%E9%85%8D%E7%BD%AE%E4%B8%8D%E4%B8%80%E8%87%B4-advancedcomponent-%E5%AF%BC%E8%87%B4)

## 行内编辑相关

### 前端拓展修改子表格数据，行内编辑保存点击不调用接口/接口入参没有修改字段的值

平台默认的行内编辑的取值逻辑如下:  
根据 `row` 数据上的`__ROW_IS_CREATED__`、`__ROW_IS_DELETED__`、`__CELL_VALUE_CHANGED__` 来判断当前行数据是否被新增/删除/修改。

> 注意这些标识有 4 个下划线

`saveHandler` 的值也是同理（需要重写保存按钮的时候，用这个方法获取入参）。

```js
let { insertRecords, updateRecords, deleteRecords } =
  tableNode?.instance?.saveHandler();
//deleteRecords:__ROW_IS_DELETED__
//insertRecords:__ROW_IS_CREATED__
//updateRecords:__CELL_VALUE_CHANGED__
```

解决：
在前端拓展处理对应逻辑时，同步改变 `row` 行数据的对应标识。  
![img](../images/01/01_row_edit_save_cell_value_changed.png)

## 前端扩展修改表格按钮禁用状态

### 方法一：直接修改节点属性（最佳实践）

```js
vm.$select("ui-view_table_toolbar_delete").data.options.disabled = true; // 工具栏按钮

// 如果是操作列按钮，可以修改 enableCondition 属性
const tableVm = vm.$select("tenant_management_menu_table_main_table");
tableVm.data.items.find(
  (i) => i.type === "operation",
).items[0].enableCondition = () => {
  return false;
};
```

### 方法二：`can` 开头钩子

`can` 开头的钩子 return `false` 即可置灰对应按钮。

| 钩子        | 控制的按钮     |
| ----------- | -------------- |
| `canCreate` | 工具栏新增按钮 |
| `canDelete` | 工具栏删除按钮 |
| `canEdit`   | 操作列编辑按钮 |

```js
hook: {
  grid: {
    // 工具栏新增按钮是否可点击
    canCreate: async (vm, params, options) => {
      return false; // 返回 false 则按钮禁用
    },
    // 工具栏删除按钮是否可点击
    // params: 选中的行数据
    canDelete: async (vm, params, options) => {
      if (params[0]?.state?.value === '1') {
        return false;
      }
      return true;
    },
    // 操作列编辑按钮是否可点击
    // params: 所处行数据
    canEdit: async (vm, params, options) => {
      if (params.rowIndex === 0) {
        return false;
      }
      return true;
    }
  }
}
```

> 子表格钩子写法相同，需嵌套在 `tabs.{field}.grid` 内，参考[子表格钩子文档](/pages/3a50a2/)。

## 表格按钮配置动态显示隐藏

> 表格按钮动态显示/隐藏分两种场景，配置方式不同。

### 操作列按钮（`buttons`）— 用 `showCondition`

针对每行的操作列按钮，使用 `showCondition` 字段，是一个函数字符串，参数为 `(row, button, vm)`，返回 `true` 显示，`false` 隐藏。

```json
{
  "type": "grid",
  "buttons": [
    {
      "name": "发布",
      "action": "xxx",
      "service": "xxx",
      "model": "xxx",
      "showCondition": "(row, button, vm) => { return row.state === '1' }"
    }
  ]
}
```

### 工具栏按钮（`tbar`）— 用 `bind_display`

针对表格顶部工具栏按钮，使用 `bind_display`，是一个模板表达式字符串，返回 `true` 显示，`false` 隐藏。

可用的上下文变量：

- `$ds.selectedTableData`：当前表格**选中行**数据数组
- `$ds.checkedDataList`：当前表格**勾选行**数据数组（需配置 `checkbox: multiple_2`）
- `$ds.checkedDataIds`：当前表格勾选数据的 id 集合

```json
{
  "action": "xxx",
  "name": "审批",
  "service": "approve",
  "model": "xxx",
  "bind_display": "${$ds.checkedDataList.length > 0 && $ds.checkedDataList[0].state.value === '0'}"
}
```

> 注意 `bind_display` 的值要用 `${ }` 包裹表达式。

## 弹窗选择器相关问题

> 弹窗选择器只能在视图配置内使用，暂不支持前端拓展内使用  
> 参考文档：[弹窗选择器](/pages/a6146b/#基础配置)

### 弹窗选择器回显异常

> **适用场景**：选择数据后，表单项不显示内容、仅显示 ID 或显示 `[object Object]`。

**配置错误**，请按照以下步骤排查：

#### 1. 排查基础组件类型

首先暂时移除该字段配置中的 `弹窗选择器` 相关配置，检查原字段的基础组件类型：

- **若原字段为 下拉框 (lookup)**：

  - **排查**：确认 `lookup` 接口返回的 `label` 是否正确。
  - **原理**：弹窗选择器在选中后，会尝试将选中的 ID 与下拉列表中的选项合并并匹配。如果下拉选项数据的 `label` 错误，会导致回显异常。
  - **解决方案**：确保 `lookup` 接口返回正确的数据结构，在不开启弹窗的情况下，该下拉框能正常回显。

- **若原字段为 输入框 (input)**：
  - **排查**：检查是否配置了 `valueField` 和 `labelField`。
  - **原理**：输入框无法解析 `key-value` 结构的对象，会将对象转为字符串，从而显示 `[object Object]` 或 ID。
  - **解决方案**：在字段配置处，将字段组件类型配置为 **下拉类型**。

```json
// 普通表单配置：
"type": "select", // 或 "lookup"
"custom": true,

// 表格行内编辑使用的弹窗选择器配置：
"editConfigs": {
  "editType": "select"
}
```

#### 2. 检查联动回填配置 (setFormValues)

如果组件类型正确但回显依然异常，请检查联动逻辑：

- **严禁自引用** : `setFormValues` 中 **不得包含当前字段本身**。  
  否则会导致数据重复赋值或触发死循环，最终导致回显失败。  
  若不需要联动更新其他字段，请务必移除以下项：

  - `setFormValues`
  - `setFormLabels`
  - `getTableValues`

  > 注意：上述配置项仅用于回填**非当前项**的其他表单字段。  
  > 参考文档：[弹窗选择器-回填其他表单项](/pages/a6146b/#回填其他表单项)

### 弹窗选择器需要 change 事件触发自定义逻辑

> 弹窗选择器没有提供 change 事件，需更换其他写法实现

- 使用弹窗选择器提供的 `confirm` 和 `customClear` 事件，分别作为 确认按钮 和 清除按钮 的触发逻辑。
- 前端拓展监听表单项数据变更，并执行自定义逻辑。

## openView 弹窗获取选中数据

### 弹窗选择器获取选中数据 【最佳实践】

在 `customSelect.useOpenView` 内配置 `confirm`，`result` 为选中数据。

```js
{
  "name": "fieldName",
  "useCustomClick": true,
  "view": {
    "customClick": {
      "customSelect": {
        "useOpenView": {
          "preId": "xxx_",
          "model": "xxx",
          "type": "grid,search",
          "checkbox": "single",
          "valueField": "id",
          "labelField": "name",
          "confirm": "(vm, result) => { console.log('选中数据', result) }"
        }
      }
    }
  }
}
```

### openView - 多选模式

配置 `checkbox: 'multiple'`，通过 `confirm` 事件内 `vm.$select(...)` 的 `$ds.transferSelected` 获取所有已选数据。

```js
openView: {
  showType: 'dialog',
  preId: 'example_001_',
  model: 'xxx',
  type: 'grid,search',
  checkbox: 'multiple',
  valueField: 'id',
  labelField: 'name',
  confirm: (vm) => {
    const selectedData = vm.$select(vm.$ds.idPre + 'table_main_table').$ds.transferSelected
    console.log('已选数据列表', selectedData)
  }
}
```

### openView - 单选模式

配置 `checkbox: 'single'`，通过 `confirm` 事件内 `vm.$select(...)` 的 `$ds.checkedDataList` 获取所有已选数据。

```js
openView: {
  showType: 'dialog',
  preId: 'example_001_',
  model: 'xxx',
  type: 'grid,search',
  checkbox: 'single',
  valueField: 'id',
  labelField: 'name',
  confirm: (vm) => {
    const selectedData = vm.$select(vm.$ds.idPre + 'table_main_table').$ds.checkedDataList
    console.log('已选数据列表', selectedData)
  }
}
```

## 本地开发环境，前端拓展不生效

> 从以下几个方向去排除

1. 检查前端项目 `\config\apps.json` 文件内 `self` 和 `apps` 配置，确认是否引入了本地的 app
2. 检查前端项目 `\apps\对应app\config\app.json` 文件内，是否配置了[按需加载](/pages/a62bee/#扩展应用按需加载)，且页面路径是否匹配
3. 项目内全局搜索不生效的前端拓展的拓展名，检查是否存在重名拓展
4. 项目内全局搜索不生效的前端拓展的拓展元素 id，检查是否存在对同个元素拓展，导致拓展覆盖或者逻辑冲突

## 视图默认查询如何配置自定义接口

### 标准视图（在线视图配置）

使用 `searchByMainTable` 配置，与 `type: 'grid'` 同级：

```json
{
  "type": "grid",
  "searchByMainTable": {
    "model": "YourModel",
    "app": "your-app",
    "service": {
      "search": "customSearchMethod",
      "count": "customCountMethod"
    },
    "args": {
      "filter": [["name", "like", "33"]]
    }
  }
}
```

子表放在 `tabs[].searchByMainTable`（与 `header` 同级）。

### openView 弹窗场景

使用 `api.grid.search`，在 `params` 内指定 `model`、`service`：

```js
openView: {
  preId: 'example_001_',
  model: 'example_unit_fmk',
  type: 'grid,search',
  api: {
    grid: {
      search: {
        params: {
          model: 'OtherModel',     // 指定其他 model
          service: 'customSearch', // 指定自定义 service 方法
          args: {
            filter: [['id', '=', 'xxx']],
            limit: 0
          }
        },
        reqPrep: (t, params) => { return params; },   // 请求前处理
        reqAfter: (vm, res) => { return res; }         // 响应后处理
      }
    }
  }
}
```

> `count` 不单独配置时，默认复用 `search` 的配置。

### 树视图

使用 `searchConfig.service` 配置：

```json
{
  "type": "tree",
  "searchConfig": {
    "model": "",
    "service": "",
    "args": {
      "filter": [["id", "=", "1234"]]
    }
  }
}
```

## 非 ER 主从表，子表自定义查询 filter 中如何引用主表选中行数据

在子表的 `tabs[].searchByMainTable` 中，通过 **`$row.字段名`** 模板变量引用主表当前选中行的字段值。

> 关键：非 ER 关系的子表必须配置 `"autoSearch": true`，才能与主表选中行联动查询。

`searchByMainTable` 配置在 `tabs[]` 的 `header` **同级**：

```json
{
  "type": "form",
  "tabs": [
    {
      "header": "子表标题",
      "searchByMainTable": {
        "autoSearch": true,
        "model": "YourSubModel",
        "app": "your-app",
        "service": {
          "search": "customSearch",
          "count": "customCount"
        },
        "args": {
          "filter": [
            ["main_id", "=", "$row.id"],
            ["org_id", "=", "$row.org_id"]
          ]
        }
      },
      "body": {
        "type": "grid,search",
        "field": "subList",
        "columns": ["name", "code"]
      }
    }
  ]
}
```

### `$row` / `$tree` 模板变量说明

| 变量          | 说明                                     |
| ------------- | ---------------------------------------- |
| `$row.字段名` | 上下表格场景中，主表格当前选中行的字段值 |
| `$tree.id`    | 左侧树视图中当前选中节点的字段值         |

### 与 ER 子表的区别

|                     | ER 子表       | 非 ER 子表                  |
| ------------------- | ------------- | --------------------------- |
| 触发查询            | 平台自动关联  | 需配置 `"autoSearch": true` |
| filter 引用主表数据 | `$row.字段名` | `$row.字段名`（相同）       |

## 视图配置 grid 限制 search 接口调用

### 主表禁用自动搜索

- 进入主表格时，不自动搜索，需要手动点击搜索按钮进行搜索。

```json
{
  "type": "grid",
  "disabledInitAutoSearch": true
}
```

### tab 配置子表或者孙表是否根据条件触发查询

- tabs 配置"controlTableSearchFn" 方法， return true 表示允许查询，false 表示不查询
- (vm, row) => 出参： vm 当前子表或者孙表 id， row 父级当前行数据
- 默认情况下，在 `return false (不查询)` 时，会清空表格数据，配置 `disabledSubTableClearOnSearch: true` 可跳过表格清空逻辑

```js
{
  "type": "form",
  "tabs": [
    {
      "header": "用户",
      "controlTableSearchFn": (vm, row) => { if (row.id == 'xxxx'){return false} else {return true} },
      "disabledSubTableClearOnSearch": true,
      "body": {
        "type": "grid,search",
        "field": "user_ids",
        "columns": [
          "login",
          "name"
        ]
      }
    }
  ]
}
```

### 搜索栏下拉框选中后禁止自动查询

> 默认情况下，搜索栏的下拉框（select/lookup）在选择选项后会立即触发查询。  
> 若需要用户手动点击搜索按钮后再执行查询，可通过 `noDirectSearch` 配置关闭此行为。
> 无视图级配置，只支持全局配置

在 `apps.json` 中添加全局配置：

```json
{
  "noDirectSearch": true
}
```

配置后，下拉框选中不再自动触发查询，需手动点击搜索按钮。

## 表格配置多个操作列

> 平台视图配置只支持单一操作列。如需在表格中展示多个操作按钮列，需通过**前端扩展**实现。

通过扩展类型 `type: 'custom'` + `beforeOperate` 钩子，将目标列改造为 `container` 容器，在其 `items` 中放置多个按钮。

**步骤：**

1. 定位目标列单元格的元素 `id`。
2. 在前端扩展中，用 `selector.value` 指向该列 `id`，在 `beforeOperate` 内修改其 `type` 和 `items`。

```js
table_column_test_extend_view: {
  selector: {
    attr: 'id',
    value: 'tenant_management_menu_table_main_table_column_code' // 目标列元素 id
  },
  type: 'custom', // 不做默认处理，由 beforeOperate 自行构造
  beforeOperate: (app, config, options) => {
    // options.element 为当前选中的列视图对象
    options.element.type = 'container'; // 将该列改为容器
    options.element.items = [
      {
        type: 'button',
        value: '按钮1',
        view: {
          click: {
            // 自定义点击逻辑
          }
        }
      },
      {
        type: 'button',
        value: '按钮2',
        view: {
          click: {
            // 自定义点击逻辑
          }
        }
      }
    ];
  },
  view: {}
}
```

> `selector.value` 中列元素的 id 规律为：`{idPre}table_main_table_column_{字段名}`。

## 如何配置 IP 地址直接访问 IIDP 平台首页（无路由后缀）？ ⭐ 最佳推荐

通过配置 Nginx，将根路径直接指向 IIDP 平台文件，无需使用/iidp 路由后缀：

### 步骤 1：配置 Nginx 直接指向根路径

在 Nginx 配置文件中添加以下配置：

```nginx
server {
    listen       80;  # 或您的实际端口
    server_name  localhost;

    # 前端文件存放路径（IIDP平台文件直接放在根目录）
    set $basePath "/xxxfront/";

    # ========== 关键配置：根路径直接指向IIDP平台 ==========
    location / {
        root $basePath;
        index  index.html index.htm;

        # 前端路由fallback处理
        if (!-e $request_filename) {
            rewrite ^/(.*) /index.html last;
            break;
        }

        # HTML文件禁用缓存
        if ($request_filename ~* .*\.(?:htm|html)$) {
            add_header Cache-Control "no-cache, no-store";
        }

        add_header Access-Control-Allow-Origin *;
        add_header Access-Control-Allow-Methods 'GET, POST, OPTIONS';
    }

    # 静态资源配置
    location ^~/docs {
        root $basePath;
    }

    location /config {
        root $basePath;
    }

    location /umdComps {
        root $basePath;
    }

    # 后端API代理
    location ^~/api/ {
        proxy_http_version 1.1;
        client_max_body_size 200m;
        proxy_pass http://后端IP:后端端口/;
    }
}
```

### 步骤 2：配置登录跳转指定菜单

在系统配置中设置默认菜单，确保登录后跳转到正确页面：

```js
// 系统级变量
// 在 apps.json 文件中配置的变量，整个系统中生效
{
    "defaultMenu": "/",  // 登录后跳转到根路径
    "routerBase"： "/"
}
```

### 验证效果

配置完成后，直接访问：

- `http://192.168.1.100:8080/` → 直接打开 → IIDP 平台首页

这样用户只需输入 IP 地址和端口即可直接访问 IIDP 平台首页，**无需任何路由后缀**。

## 前端拓展追加节点的位置错误

前端拓展追加节点的位置错误通常是由于选择器（selector）配置不当或节点 ID 动态变化导致的。  
以下是具体原因和解决方案：

### 一、常见原因

#### 1. 选择器（selector）配置错误

- **value 值不正确**：selector.value 指定的元素 ID 在当前页面中不存在或已变化。
- **type 值不匹配**：type 配置（before、after、replace、append 等）与期望的插入位置不符。


#### 2. 节点 ID 动态变化

- **OpenView 场景**：OpenView 打开的页面，其元素 ID 会被添加 preId 前缀，导致基于原始 ID 的扩展失效。
- **菜单 Name 被修改**：如果后端修改了菜单种子数据的 name，会导致前端生成的 ID 变化。

#### 3. 扩展执行顺序问题

- 多个扩展对同一节点进行操作时，执行顺序可能导致最终位置不符合预期。

### 二、解决方案

#### 1. 正确选择扩展位置

**步骤**：

1. 进入目标页面，按 F12 打开开发者工具。
2. 使用元素选择器，找到需要扩展的节点，并记录其 id。
3. 确保扩展配置中的 selector.value 与该 id 完全一致。

**示例

```js
export default {
  book_borrow_management_extend_view: {
    type: 'before', // 在选中节点前插入
    selector: {
      attr: 'id',
      value: 'demo_books_borrow_management_menu_main_table_search_container' // 从页面实际获取的ID
    },
    view: { ... }}}

``` 

#### 2. 选择正确的 type

根据您希望插入的位置，选择对应的 type：

|类型|	说明|
| ---- | ---- |
|before|	在选中节点前面插入视图|
|after	|在选中节点后面插入视图|
|merge|	选中节点与新 view 视图进行属性合并 若选中节点有 items 则不合并 items 属性其他属性都深度合并|
|unshift|	在选中节点的 items 头部插入视图|
|replace|	替换选中节点为视图节点|
|append|	在选中节点的 items 尾部插入视图|
|delete|	删除选中节点|
|custom|	对选中节点不做任何处理但会执行 beforeOperate 函数，用户自行动态构造|


#### 3 确保扩展加载顺序正确

- 扩展按扩展名字符串的字母数字顺序（0-9, a-z）执行。
- 后执行的扩展可以覆盖先执行的扩展。
- 如果需要控制顺序，可以通过命名调整（如 01_xxx、02_xxx）。
- 配置了extend 继承的父扩展会优先执行


## 表单项值为空时设置为 null

> 数据类型为非 `'auto'`、`'string'`、`'text'` 的表单项，用户未输入时默认值为空字符串，如需设置为 `null`，可使用以下方法处理。

使用 `vm.$cmd.meta.formFormat.handleEmptyData` 方法：

```js
const formFormatData = vm.$cmd.meta.formFormat.handleEmptyData(
  vm,
  formData,
  fields
);
```

| 属性名 | 说明 | 类型 |
| ------ | ---- | ---- |
| `vm` | 当前视图实例 | Object |
| `formData` | 表单数据 | Object |
| `fields` | loadView 接口返回的 fields | Object |

返回处理后的 `formData` 对象，将非 `'auto'`、`'string'`、`'text'` 数据类型的空值设置为 `null`。

## 三级联动实现

### 方式一：在线视图 `linkage` 配置（最佳实践）

适用于**在线视图**（search / form / grid）。通过 `linkage.children` 声明子级字段，`linkage.params` 将当前选中值传给下级接口。

```js
// 以"省 → 市 → 区"为例（form 视图）
{
  "body": {
    "columns": [
      {
        "label": "省", "name": "province", "widget": "select",
        "linkage": {
          "params": {},
          "children": ["city", "area"]   // 声明联动的子字段
        }
      },
      {
        "label": "市", "name": "city",
        "linkage": {
          "params": { "provinceId": "${province}" }, // 接收上级值
          "children": ["area"]
        }
      },
      {
        "label": "区", "name": "area",
        "linkage": {
          "params": {
            "provinceId": "${province}",
            "cityId": "${city}"
          }
        }
      }
    ],
    "type": "form"
  }
}
```


### 方式二：非在线视图 `ds_config + bind_filter`

适用于非在线视图，利用 `ds_config.doWhenChange: 'reload'` + `bind_filter` 绑定上级表单值，值变化时自动重新请求下级数据。

```js
{
  type: 'form',
  dataSource: {
    form: { province: '', city: '', area: '' }
  },
  items: [
    {
      type: 'select', text: '省', name: 'province',
      ds_config: {
        type: 'meta', name: 'data1',
        autoRequest: true,
        options: { params: { args: { properties: ['name'] }, service: 'search', model: 'xxx' } }
      },
      bind_options: '$ds.data1'
    },
    {
      type: 'select', text: '市', name: 'city',
      ds_config: {
        type: 'meta', name: 'data2',
        doWhenChange: 'reload',  // 上级变化时重新加载
        options: {
          params: {
            args: {
              bind_filter: '${[["province_id", "=", $ds.form.province]]}', // 绑定省的值
              properties: ['name']
            },
            service: 'search', model: 'xxx'
          }
        }
      },
      bind_options: '$ds.data2'
    },
    {
      type: 'select', text: '区', name: 'area',
      ds_config: {
        type: 'meta', name: 'data3',
        doWhenChange: 'reload',
        options: {
          params: {
            args: {
              bind_filter: '${[["city_id", "=", $ds.form.city]]}', // 绑定市的值
              properties: ['name']
            },
            service: 'search', model: 'xxx'
          }
        }
      },
      bind_options: '$ds.data3'
    }
  ]
}
```

> `bind_filter` 内通过 `$ds.form.字段名` 引用当前表单数据，`doWhenChange: 'reload'` 保证上级值变更时下级自动刷新。



## 图标字体乱码排查

> 图标显示为方块、问号或奇怪字符时，按以下顺序排查。

### 1. 字体文件加载失败（优先排查）

打开浏览器 DevTools → **Network** 标签，过滤 `.woff2` / `.woff` / `.ttf`，确认字体文件是否 404 或请求失败。

常见原因：
- 字体文件路径与部署路径不匹配
- 字体文件未被打包到构建产物中

### 2. `font-family` 名称不一致

对比 `@font-face` 定义与使用处的 `font-family` 值是否完全一致（大小写敏感）：

```css
/* 定义 */
@font-face {
  font-family: "iconfont";
  src: url('iconfont.woff2') format('woff2');
}

/* 使用 —— 必须与上面完全一致 */
.iconfont {
  font-family: "iconfont" !important;
}
```

### 3. Unicode 编码被转义

在 DevTools → **Elements** 中检查图标元素的 `content` CSS 属性，确认 Unicode 字符未被错误处理成文本字符串（如 `\ue001` 而非实际字符）。

### 4. CSS 文件编码问题

确认 CSS 文件保存时为 **UTF-8（无 BOM）** 编码。IDEA 可在 `File → Settings → Editor → File Encodings` 统一设置。

### 5. 字体资源跨域被拦截

DevTools → **Console** 查看是否有 CORS 错误（`Cross-Origin Request Blocked`）。需要服务器对字体文件添加 `Access-Control-Allow-Origin` 响应头。

### 6. iconfont.js 加载时序问题（Symbol 方式）

如使用 SVG Symbol 方式（引入 `iconfont.js`），确认 DevTools → Network 中该脚本加载成功且无报错。


## 封装通用下拉树表格组件业务组件：


**1. 组件文件位置**
```

或
```

**2. 组件实现代码**
```vue
<template>
  <div class="tech-select-tree-table">
    <div class="select-trigger" @click="openDialog">
      <el-input
        v-model="displayValue"
        :placeholder="placeholder"
        readonly
        suffix-icon="el-icon-arrow-down"
      />
    </div>
    
    <el-dialog
      :visible.sync="dialogVisible"
      :title="title"
      width="70%"
      :before-close="handleClose"
    >
      <div class="dialog-content">
        <div class="tree-section">
          <el-tree
            ref="tree"
            :data="treeData"
            :props="treeProps"
            :load="loadNode"
            lazy
            highlight-current
            @node-click="handleNodeClick"
          />
        </div>
        
        <div class="table-section">
          <el-table
            ref="table"
            :data="tableData"
            highlight-current-row
            @current-change="handleTableSelect"
          >
            <el-table-column
              v-for="col in columns"
              :key="col.prop"
              :prop="col.prop"
              :label="col.label"
              :width="col.width"
            />
          </el-table>
          
          <el-pagination
            :current-page="page.current"
            :page-size="page.size"
            :total="page.total"
            layout="prev, pager, next"
            @current-change="handlePageChange"
          />
        </div>
      </div>
      
      <div slot="footer">
        <el-button @click="dialogVisible = false">取消</el-button>
        <el-button type="primary" @click="confirmSelect">确定</el-button>
      </div>
    </el-dialog>
  </div>
</template>

<script>
export default {
  name: 'tech-select-tree-table',
  
  props: {
    value: { type: [String, Number, Array], default: '' },
    placeholder: { type: String, default: '请选择' },
    title: { type: String, default: '选择数据' },
    treeProps: {
      type: Object,
      default: () => ({ label: 'name', children: 'children', isLeaf: 'isLeaf' })
    },
    columns: {
      type: Array,
      default: () => [
        { prop: 'code', label: '编码', width: 120 },
        { prop: 'name', label: '名称', width: 150 }
      ]
    },
    treeApi: { type: Object, required: true },
    tableApi: { type: Object, required: true },
    valueField: { type: String, default: 'id' },
    labelField: { type: String, default: 'name' },
    multiple: { type: Boolean, default: false }
  },
  
  data() {
    return {
      dialogVisible: false,
      displayValue: '',
      treeData: [],
      tableData: [],
      selectedRow: null,
      currentTreeNode: null,
      page: { current: 1, size: 10, total: 0 }
    };
  },
  
  watch: {
    value: {
      immediate: true,
      handler(val) { this.updateDisplayValue(val); }
    }
  },
  
  methods: {
    openDialog() {
      this.dialogVisible = true;
      this.loadTreeData();
    },
    
    async loadTreeData() {
      const res = await window.Tech.httpMeta({ data: this.treeApi });
      this.treeData = res.data || [];
    },
    
    async loadNode(node, resolve) {
      if (node.level === 0) return resolve(this.treeData);
      const params = {
        ...this.treeApi,
        data: {
          params: {
            ...this.treeApi.data.params,
            args: { ...this.treeApi.data.params.args, parent_id: node.data.id }
          }
        }
      };
      const res = await window.Tech.httpMeta({ data: params });
      resolve(res.data || []);
    },
    
    handleNodeClick(data) {
      this.currentTreeNode = data;
      this.page.current = 1;
      this.loadTableData();
    },
    
    async loadTableData() {
      const params = {
        ...this.tableApi,
        data: {
          params: {
            ...this.tableApi.data.params,
            args: {
              ...this.tableApi.data.params.args,
              parent_id: this.currentTreeNode?.id,
              limit: this.page.size,
              offset: (this.page.current - 1) * this.page.size
            }
          }
        }
      };
      const res = await window.Tech.httpMeta({ data: params });
      this.tableData = res.data || [];
      this.page.total = res.count || 0;
    },
    
    handleTableSelect(row) { this.selectedRow = row; },
    
    handlePageChange(page) {
      this.page.current = page;
      this.loadTableData();
    },
    
    confirmSelect() {
      if (!this.selectedRow) {
        window.ELEMENT.Message.warning('请选择数据');
        return;
      }
      const value = this.selectedRow[this.valueField];
      const label = this.selectedRow[this.labelField];
      this.displayValue = label;
      this.dialogVisible = false;
      this.$emit('input', value);
      this.$emit('change', { value, label, row: this.selectedRow });
      return { value, label, row: this.selectedRow };
    },
    
    async updateDisplayValue(val) {
      if (!val) { this.displayValue = ''; return; }
      const params = {
        ...this.tableApi,
        data: {
          params: {
            ...this.tableApi.data.params,
            args: { ...this.tableApi.data.params.args, filter: [[this.valueField, '=', val]], limit: 1 }
          }
        }
      };
      const res = await window.Tech.httpMeta({ data: params });
      if (res.data?.[0]) this.displayValue = res.data[0][this.labelField];
    },
    
    handleClose() { this.dialogVisible = false; }
  }
};
</script>
```
