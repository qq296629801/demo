# 02 — 软件需求规格（SRS）

## 1. 引言

### 1.1 目的

本 SRS 描述 SIE IIDP Backend Demo 工程的功能需求。该工程定位为 **学习示例工程**，覆盖 IIDP 平台后端原子能力，供新手开发者了解并实践 SNEST Engine 的各类特性。

### 1.2 范围

| 模块 | 覆盖能力 |
|------|---------|
| 基础 CRUD | 创建、查询、更新、删除模型记录 |
| 字段校验 | `@Validate` 注解驱动的字段约束 |
| 逻辑删除 | `isLogicDelete=True` 软删除模式 |
| 自动审计 | 创建/更新时间、操作人自动填写 |
| 启用/禁用 | 批量状态切换 |
| Excel 导入/导出 | EasyExcel 集成，支持模板导入与过滤导出 |
| 下拉选项 | `@Selection/@Option` 枚举选项绑定 |
| 树形结构 | 省-市-区-街道四级区域树 |
| 继承/扩展（MBM） | 子类 Model 覆盖父类方法、插槽扩展 |
| 定时任务 | XXL-JOB 执行器接入 |
| 服务注入 | `@InjectMeta` 将 Service 注入 Model |
| 分片存储 | 按月分表策略 |

## 2. 功能需求

### FR-001 区域管理

**优先级**: 高

| # | 功能点 | 描述 |
|---|--------|------|
| FR-001-01 | 区域 CRUD | 支持新增、查询、更新、删除区域记录 |
| FR-001-02 | 字段校验 | `areaCode`（非空，唯一）、`areaName`（非空，≤60字）、`areaType`（非空，省/市枚举）、`isEnable`（非空）|
| FR-001-03 | 批量启用/禁用 | 根据 ID 列表批量切换 `isEnable` 字段 |
| FR-001-04 | Excel 导出 | 按 Filter 导出区域数据（`areaCode`、`areaName`）为 `.xlsx` |
| FR-001-05 | Excel 导入 | 从 Sheet1 导入区域数据，编码不超过 6 位 |
| FR-001-06 | 省市下拉列表 | 按 `areaType` 过滤返回下拉选项 |
| FR-001-07 | 城市联动查询 | 按 `parentCode` 查询下级地区，返回按编码升序 |
| FR-001-08 | 按月分表 | 区域数据按月分片存储 |

### FR-002 学生管理

**优先级**: 高

| # | 功能点 | 描述 |
|---|--------|------|
| FR-002-01 | 学生 CRUD | 支持学生信息的完整 CRUD 操作 |
| FR-002-02 | 关联班级 | `classId` 关联 `example_class` 模型 |
| FR-002-03 | 学生档案 | 独立的学生档案模型，含逻辑删除和自动审计 |

### FR-003 物料管理

**优先级**: 中

| # | 功能点 | 描述 |
|---|--------|------|
| FR-003-01 | 物料 CRUD | 物料基本资料的增删改查 |
| FR-003-02 | 物料分类树 | 支持父子层级的物料分类 |
| FR-003-03 | 物料扩展属性 | 物料与属性一对多关联 |
| FR-003-04 | 物料打印设置 | 每个物料可配置打印模板 |

### FR-004 物料库存（MBM 继承示例）

**优先级**: 中

| # | 功能点 | 描述 |
|---|--------|------|
| FR-004-01 | 继承 BaseInventory | `MbmMaterialInventory` 继承 `demo_material_inventory`，扩展 `maxCount`、`mbmMinCount` |
| FR-004-02 | 新增记录 | 支持批量新增物料库存 |
| FR-004-03 | 修改记录 | 入参出参均可扩展 |
| FR-004-04 | 插槽2扩展 | `slot2` 为二开方提供业务逻辑注入点 |
| FR-004-05 | 焊接工序覆盖 | `weld` 覆盖父类焊接，跳过装配步骤 |

### FR-005 定时任务（XXL-JOB）

**优先级**: 低

| # | 功能点 | 描述 |
|---|--------|------|
| FR-005-01 | 执行器注册 | XXL-JOB Executor 自动注册到调度中心 |
| FR-005-02 | 健康检查 | `GET /sayHello` 返回固定字符串 |

## 3. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | SNEST Engine ORM 查询响应 < 200ms（单表，索引完善） |
| 并发 | Spring Boot Tomcat 默认线程池，支持标准并发 |
| 安全 | IAM 权限集成，接口级权限控制 |
| 可维护性 | 模型即文档，`@Property(displayName)` 即字段说明 |
| 可扩展性 | MBM 继承模式允许二开方安全覆盖/扩展业务逻辑 |

---

## 4. 约束

1. 代码编写必须遵循 SNEST SDK 规范（注解驱动，不直接操作 SQL）
2. 字段命名遵循驼峰（Java）→ 下划线（数据库）自动映射
3. 分片表不支持物理删除，建议配合逻辑删除使用
