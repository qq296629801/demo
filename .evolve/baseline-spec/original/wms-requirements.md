# WMS 仓库管理系统 — 需求规格说明书（基础版）

**文档版本**：v1.0  
**业务领域**：WMS（Warehouse Management System）  
**来源**：基于 ModernWMS 开源项目（https://github.com/fjykTec/ModernWMS）功能整理，结合行业通用 WMS 规范编写  
**范围**：SKU 商品管理 + 入库单（ASN）管理 两个核心模块

---

## 1. 功能模块清单

| 模块编号 | 模块名称 | 描述 |
|---------|---------|------|
| M-01 | SKU 商品管理 | 管理仓库内所有商品的基础信息，包括编码、名称、单位、分类、库存上下限 |
| M-02 | 入库单（ASN）管理 | 管理采购/调拨入库的单据，从创建到收货完成的全流程 |

---

## 2. 用户故事

### M-01 SKU 商品管理

**US-001** 作为仓库管理员，我想要新增商品 SKU 信息，以便后续入库和出库操作能够关联到具体商品。

- AC-001-01：商品编码（sku_code）在系统内唯一，重复时拒绝保存并提示"编码已存在"
- AC-001-02：商品名称（sku_name）不能为空，最大长度 200 字符
- AC-001-03：单位（unit）不能为空
- AC-001-04：库存下限（safety_stock_qty）不能为负数，默认为 0
- AC-001-05：保存成功后，商品列表立即可见新增记录

**US-002** 作为仓库管理员，我想要查询商品列表，支持按编码或名称模糊搜索，以便快速定位商品。

- AC-002-01：搜索结果按创建时间倒序排列
- AC-002-02：支持分页，默认每页 20 条

**US-003** 作为仓库管理员，我想要修改商品的基础信息（名称、单位、安全库存），以便维护商品数据准确性。

- AC-003-01：sku_code 创建后不可修改
- AC-003-02：有库存的商品，修改单位时系统警告"该商品有在库库存，修改单位可能影响数据一致性"，需二次确认

---

### M-02 入库单（ASN）管理

**US-004** 作为采购员，我想要创建入库单，填写预计到货商品和数量，以便仓库提前做好收货准备。

- AC-004-01：入库单号（asn_no）由系统自动生成，格式 `ASN-YYYYMMDD-XXXXX`（XXXXX 为5位流水号）
- AC-004-02：入库单创建时状态为"待收货"（status=0）
- AC-004-03：入库单明细至少包含 1 行商品
- AC-004-04：同一入库单中同一 SKU 不允许出现两行（需合并）

**US-005** 作为仓库收货员，我想要对入库单进行收货操作，记录实际收到的数量，以便更新库存。

- AC-005-01：收货数量不能超过入库单明细的预计数量
- AC-005-02：部分收货后，入库单状态变为"部分收货"（status=1）
- AC-005-03：全部明细收货完成后，入库单状态自动变为"已完成"（status=2）
- AC-005-04：收货完成后，对应 SKU 的在库数量（qty_available）相应增加

**US-006** 作为仓库管理员，我想要取消状态为"待收货"的入库单，以便处理错误录入或需求变更的情况。

- AC-006-01：只有状态为"待收货"（status=0）的入库单可以取消
- AC-006-02：已取消的入库单不可恢复，不可收货

## 3. 数据模型

### 3.1 SKU 商品表（wms_sku）

| 字段名 | 类型 | 必填 | 唯一 | 说明 |
|--------|------|------|------|------|
| id | BIGINT | 是 | 是 | 主键，自增 |
| sku_code | VARCHAR(64) | 是 | 是 | 商品编码，创建后不可修改 |
| sku_name | VARCHAR(200) | 是 | 否 | 商品名称 |
| unit | VARCHAR(20) | 是 | 否 | 计量单位（个/箱/kg等） |
| category | VARCHAR(64) | 否 | 否 | 商品分类 |
| safety_stock_qty | DECIMAL(18,4) | 否 | 否 | 安全库存下限，默认0，不能为负 |
| is_valid | TINYINT | 是 | 否 | 是否启用：1-启用，0-停用，默认1 |
| remark | VARCHAR(500) | 否 | 否 | 备注 |
| create_time | DATETIME | 是 | 否 | 创建时间 |
| update_time | DATETIME | 是 | 否 | 最后修改时间 |

### 3.2 入库单主表（wms_asn_master）

| 字段名 | 类型 | 必填 | 唯一 | 说明 |
|--------|------|------|------|------|
| id | BIGINT | 是 | 是 | 主键，自增 |
| asn_no | VARCHAR(32) | 是 | 是 | 入库单号，系统生成，格式 ASN-YYYYMMDD-XXXXX |
| status | TINYINT | 是 | 否 | 状态：0-待收货，1-部分收货，2-已完成，9-已取消 |
| expected_arrival_date | DATE | 否 | 否 | 预计到货日期 |
| remark | VARCHAR(500) | 否 | 否 | 备注 |
| create_time | DATETIME | 是 | 否 | 创建时间 |
| update_time | DATETIME | 是 | 否 | 最后修改时间 |

### 3.3 入库单明细表（wms_asn_detail）

| 字段名 | 类型 | 必填 | 唯一 | 说明 |
|--------|------|------|------|------|
| id | BIGINT | 是 | 是 | 主键，自增 |
| asn_id | BIGINT | 是 | 否 | 关联入库单主表 ID（ManyToOne） |
| sku_id | BIGINT | 是 | 否 | 关联 SKU 商品 ID（ManyToOne） |
| expected_qty | DECIMAL(18,4) | 是 | 否 | 预计数量，必须 > 0 |
| received_qty | DECIMAL(18,4) | 是 | 否 | 已收货数量，默认0 |
| create_time | DATETIME | 是 | 否 | 创建时间 |

**约束**：同一入库单（asn_id）中，sku_id 不允许重复。

## 4. 业务规则与约束

1. **编号生成**：入库单号 `ASN-YYYYMMDD-XXXXX`，同一天内流水号从 00001 递增，跨天重置。
2. **状态流转**：
   ```
   待收货(0) → 部分收货(1) → 已完成(2)
   待收货(0) → 已取消(9)
   ```
   其他状态流转非法，服务端拒绝并返回错误。
3. **库存更新**：收货操作是唯一可增加 `qty_available` 的途径（本文档范围内）。
4. **SKU 停用**：停用（is_valid=0）的 SKU 不能出现在新入库单明细中。
